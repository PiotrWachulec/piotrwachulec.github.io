About

Hi! My name is Piotr. Let me tell you something about me.

Technically

Currently

Azure Cloud Architect with about 3+ years of commercial experience in technologies (overall 5+ years of commercial experience in IT). Also associated with the .NET platform. Most of my recent projects were based on the Azure & C#/.NET stack.

I have experience also with plenty of Azure services: Virtual machines, App services, databases: MSSQL, PostgreSQL, MySQL, virtual networks, Azure Virtual Desktop, etc. In my daily tasks, I was often responsible for creating and setting up CI & CD processes due to the fact that I am a big fan of automatization. Depending on the client and the project I used mainly Azure DevOps.

I am a big fan of the approach that a tool should get to the problem being solved, so I am open to changes and new tools if the problem being solved requires it. #TechnologyAgnostic

The culmination of the above is my big passion for software and system architecture. Every day I am trying to learn new things in these topics.

Previous and other technical experience

I have been associated with Azure and C#/.NET environment for the last 3+, but I have experience (commercial and not) in many other technologies.

In my first professional job, I wrote programs for microprocessors that control high-power supplies. There I used C and a real-time operating system from Texas Instruments. I developed a .NET application that gathered some data from the processor and his peripherals.

In my next job, I was responsible for cooperation and development applications related to SmartTV. There I used JavaScript and NativeClient (it was a solution from Google which enables to run of native C++ code in a web browser).

Then I migrated to Java stack (Java 8, Spring, Hibernate), and I prepared with the team proof of concept of the platform that was to facilitate contact of the personal trainers with their’s pupils - direct messages, preparing training plans, observing pupil’s progress and so on. Besides the development backend part of the application, I was responsible for managing the backend developer team.

Off the clock, I worked on a battery management system for the electric cart (both software and hardware sides). As a part of my BSc thesis, I prepared an end-to-end controller for an electro-pneumatic positioner. Electropneumatic positioner is an element of an automation system that controls valve opening level regarding state other parts of the system. I designed hardware, prepared software in C where I implemented two control algorithms: PID and based on fuzzy logic and desktop software which send commands to the hardware part of the controller and gather data, so I have experience also in electronics and low-level programming.
